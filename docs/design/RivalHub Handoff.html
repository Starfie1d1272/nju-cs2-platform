使用规则

  • 背景层次只能从 bg → panel-low → panel → panel-hi 单向递进。同一组件内最多用两层 · 不要堆三层卡片。
  • 边框 border 用于结构分割 · border-hi 仅在 hover/focus 出现 · 不要把边框当背景色用。
  • 主色 accent 一屏只用一次 (一个主 CTA + 一组数据高亮)。其余用 text · text-mid 表达层级。
  • 状态色不互相替代:胜场用 ok · LIVE 用 danger · 等待中用 warn。
  • 所有颜色加透明度时用 8-bit hex 后缀 (+ "1f" ≈ 12% · + "55" ≈ 33%)。

规则

  • 所有"标签"必须 mono · 全大写 · letter-spacing 至少 0.10em。中文标签不全大写 · 但保留 mono + tracking。
  • 所有数字 (rating · ADR · 比分 · 时间 · ID) 用 mono · 永远右对齐。tabular-nums 已是 JetBrains Mono 默认。
  • 正文不用 mono。叙述性段落 / 按钮 / 表格主体内容用 Geist。
  • 展示标题用 Geist 600 · letter-spacing 负值。永远不要给中文展示标题加 letter-spacing 正值。

[ 04 ] Radii & Borders

几何接近"零圆角"。Tactical Grid 方向克制 · 不用大圆角。

RSM

  • 所有边框都是 1px solid var(--border) · 永远不要超过 1px。
  • 状态色边框用 color + "55" (≈33%) 透明度 · 配 color + "10" 填充。
  • 左侧 3px 强调色边 (border-left) 是公司唯一"装饰性"边框 · 用于 StatusBanner / 关键卡片。

[ 07 ] Composition Patterns

  • Eyebrow + Title + Sub · 所有页面顶部三件套。eyebrow 用 mono 大写 + tracking-eyebrow · title 用 display · sub 用 fg-mid。
  • Mono labels everywhere · 任何 "label" (列标题 · KV pair key · section 编号) 一律 mono 10px 大写 tracking-label。
  • Right-align numbers · 表格中所有数字列 text-right + font-mono + tabular-nums。
  • One accent per fold · 一屏首屏只允许一个 accent CTA + 一个 accent 数据高亮 (例:trending up · #1 rank)。
  • Bracket as a Card · bracket / leaderboard 行点击都跳详情页 · 不开 modal。
  • Inline confirm > Modal · 危险操作首选 InlineConfirm · Modal 仅用于跨上下文 (登录 · 上传)。
  • Live tick 用 box-shadow · box-shadow: 0 0 6px var(--danger) 给 LIVE 状态点。不要 blur 整个组件。
  • Hairline grid 在 body · 不要给单个卡片再叠网格。整页 background 一次性铺。

[ 09 ] Engineering Handoff Checklist

  • 导入字体 · 在 app/layout.tsx 用 next/font/google 加载 Geist + JetBrains Mono · 中文 fallback 留给系统。
  • 合并 Tailwind config · 把第 5 节 patch 合并进 tailwind.config.ts · 不删除 shadcn 默认变量。
  • 替换 CSS 变量 · 把第 5 节 :root 写进 globals.css 的 @layer base。
  • 建立 components 目录 · components/ui/ 放 shadcn 原件 · components/rivalhub/ 放本设计系统新组件 (Btn-wrapper · Panel · StatusBanner · Stat · MiniStat · TeamBadge · PosChip · EmptyState · ErrorState · InlineConfirm · Field-wrapper · Bracket)。
  • 页面顺序 · 建议: Landing → Login/Invite → Season home → Register → Captains → Draft → Teams/Team → Matches/Match → Stats → Player → Admin。先打通"看"再打通"操作"。
  • 实时部分 · Draft room + Live ticker 需要 WebSocket 或 SSE · Match scoreboard 可定期 poll。
  • Recharts 主题 · 写一个 chartTheme.ts 把 stroke/fill 全部映射到 CSS 变量 · 避免每个图表重复定义。
  • Mobile · 当前未做移动端 · 下一阶段需要为 Landing · Match · Stats · Player 四页做窄屏版本。
  • 命名规则 · 组件文件 PascalCase.tsx · 工具文件 kebab-case.ts · CSS 变量 kebab-case · TS 类型 PascalCase。
