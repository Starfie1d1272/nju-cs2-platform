# UI 视觉设计

> 所有页面基于深色主题，详细 token 值见 `docs/ui-tokens.md`。

---

## 通用三态规范

每个数据展示区域必须处理以下三态：

| 状态 | 展示方式 |
|---|---|
| Loading | Skeleton 占位（灰色方块，高度与实际内容一致） |
| Empty | 居中图标 + 一行说明文字 + 可选的行动按钮 |
| Error | 红色 Toast 提示 + 页面内 Error UI（不替换整页） |

## 管理后台

### `/admin/login`
标准居中登录卡片，invite code + password 两个输入框 + 登录按钮。

### `/admin/[seasonSlug]/registrations`
表格布局，列：邮箱 / 主选 / 段位 / 截图 / 状态 / 操作（通过/拒绝/等待）。

### `/admin/[seasonSlug]/captains`
两栏：左侧投票排行，右侧确认按钮组（选择前 8 名 + 手动排序 draft_order）。

### `/admin/[seasonSlug]/draft`
单页面：一键开始/暂停选秀 + 强制跳过当前队 + 实时日志。

### `/admin/[seasonSlug]/matches`
比赛列表 + 录入比分表单（两个 number input + 提交按钮）。

## 移动端断点策略

- `sm`（640px）：导航折叠为 hamburger
- `md`（768px）：报名表单单列 → 双列
- `lg`（1024px）：选秀直播间 4 列队伍网格
- 全站最小宽度：320px
